# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Layout

The actual app lives in `insync-resume-ai/` (a Lovable-generated TanStack Start project). **All commands below must be run from that subdirectory.** The repository root only contains this app and a `.venv/` (unused by the app).

## Commands

Package manager is **Bun** (`bun.lock`, `bunfig.toml`). Do not introduce npm/pnpm/yarn lockfiles.

```sh
bun install         # install deps
bun run dev         # vite dev server
bun run build       # production build (Cloudflare Workers target)
bun run build:dev   # development-mode build
bun run preview     # preview built output
bun run lint        # eslint .
bun run format      # prettier --write .
```

No test runner is configured.

### Supply-chain guard
`bunfig.toml` sets `minimumReleaseAge = 86400` (24h). Packages published in the last day are skipped on install. The single exclusion is `@lovable.dev/vite-tanstack-config`. **Confirm with the user before adding any new entry to `minimumReleaseAgeExcludes`.**

## Architecture

### Stack
TanStack Start (SSR) + TanStack Router (file-based, code-generated tree) + React 19 + Tailwind v4 + shadcn/ui (new-york style, slate base, dark forced on `<html>`) + framer-motion. Deploys to **Cloudflare Workers** (`wrangler.jsonc`, `nodejs_compat` flag).

### The Vite config is opinionated — don't duplicate plugins
`vite.config.ts` uses `@lovable.dev/vite-tanstack-config`'s `defineConfig`. That wrapper already includes:
- `tanstackStart`, `viteReact`, `tailwindcss`, `tsConfigPaths`, `cloudflare` (build-only)
- `componentTagger` (dev-only), `VITE_*` env injection, `@/*` alias, React/TanStack dedupe
- error logger plugins, sandbox detection (port/host/strictPort)

Adding any of these manually will break the app with duplicate-plugin errors. Pass extra config via `defineConfig({ vite: { ... } })`.

### SSR error handling (load-bearing)
TanStack Start runs on Cloudflare Workers via h3. h3 silently converts in-handler throws into a generic 500 JSON response (`{"unhandled":true,"message":"HTTPError"}`), so a plain try/catch around the server entry never sees them. The recovery pipeline:

1. `src/lib/error-capture.ts` — global `error`/`unhandledrejection` listeners stash the last error (5s TTL) in a module-scoped slot.
2. `src/server.ts` — the real Worker entry. It dynamically imports `@tanstack/react-start/server-entry`, then sniffs the response: if it's a 500 with the exact h3-swallowed shape, it logs `consumeLastCapturedError()` (the real stack) and replaces the body with the branded `renderErrorPage()`.
3. `src/start.ts` — registers a `requestMiddleware` (`errorMiddleware`) that catches non-`statusCode` errors before they reach h3 and returns the branded error page directly.

`wrangler.jsonc` `main` alone is **not** enough to redirect the SSR entry — `@cloudflare/vite-plugin` builds from `tanstackStart.server.entry`, which is set to `"server"` in `vite.config.ts`. If you rename `src/server.ts`, update both.

### Routing
File-based routes in `src/routes/` (`__root.tsx`, `index.tsx`). `src/routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` — never edit by hand; it's excluded from prettier. Root route is `createRootRouteWithContext<{ queryClient: QueryClient }>` and `__root.tsx` provides the `QueryClientProvider`, head metadata, and the 404/error components.

### Current product surface
The app is essentially one page: `src/components/ResumeScreener.tsx` driven by mocked data in `src/lib/mock-data.ts` (no backend wired up yet). Stages: `input → loading → results`.

### Conventions
- Path alias `@/*` → `./src/*` (set in `tsconfig.json` and re-exposed by the Vite wrapper).
- shadcn/ui components live in `src/components/ui/` (46 components installed). Aliases in `components.json`: `@/components`, `@/lib/utils`, `@/components/ui`, `@/hooks`.
- Tailwind v4 uses CSS-first config in `src/styles.css` (`@theme inline { ... }`). Custom semantic colors (`emerald`, `mint`, `warn`, `danger`, `surface`, `hairline`) are defined as CSS vars there — prefer them over hard-coded colors.
- Prettier: `printWidth: 100`, double quotes, semis, trailing commas.
- ESLint blocks `import "server-only"` (Next.js convention). For server-only modules, use the `*.server.ts` suffix or `@tanstack/react-start/server-only`.
